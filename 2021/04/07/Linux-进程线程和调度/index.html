Linux 进程线程和调度

发表于 2021-04-07 分类于 Linux
本文字数： 5.1k 阅读时长 ≈ 5 分钟

进程控制块PCB

进程是资源封装的单位（pid，内存、文件系统、文件、信号、信号处理函数）

  • pid是数量是有限的
  • fork炸弹

task_struct 管理，包含三种数据结构

  • 链表：task_struct -> task_struct -> task_struct
  • 树：保存父子关系
  • hash：pid -> task_struct

生命周期

  • 深度睡眠：只有等到资源才会唤醒，不响应任何信号，kill -9 都杀不掉，比如发生pagefault的时候
  • 浅度睡眠：等到资源或者有信号异步打断

僵尸进程：

  • 资源已经释放：无内存泄漏
  • task_struct还在：父进程可以查到子进程的死因（比如检测到子进程是被哪个信号杀死的）

暂停态：作业控制的方式，睡眠是自己主动的，暂停是基于信号被动的
